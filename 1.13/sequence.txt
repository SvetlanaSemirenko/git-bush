1. открывем Git Bash
2. проверяем в какой папке находимся pwd
3. переходим в папку с проектом cd GIT/Git_Bash
4. посмотреть содержимое папки ls -la
5. проверка наличия git репозитория git status 
Сообщение: " not a git repository (or any of the parent directories)" означает, что в этой папке нет git repository
6. создание git repository git init
Сообщение:"Initialized empty Git repository in C:/Users/ssvet1/GIT/Git_Bash/.git/" означаеи инициализацию пустого репозитория в папке 
В ней хранится вся информация об истории изменений в текущем репозитории
7. После введения git status все файлы и папки выделятся красным цветом, это значит, что они новые для репозитория
8. Добавление всех новых папок в репозиторий git add .
9. продвижение их в репозиторий git commit -m 'initiai', что значит первый коммит после создания репозитория
10. git status покажет On branch master
nothing to commit, working tree clean , что означает, что несохранённых изменений нет
11. git show покажет информацию о последнем коммите и его содержимом
12. q выход из режима просмотра.
13. посмотреть историю коммитов можно git log
14. чтобы доавить в индекс не все, что было сделано, можно прописать git add 1.13/sequence.txt
Тогда лобавятся только изменения внесённые в файл sequence.txt 
15. git status выведет их зелёным цветом, а созданный, но не добавленный test.html попадет в Untracked files, т.е. те, 
что не отсдеживаются системой контроля версий GIT. 
16. Некоторые файлы не рекомендуется сохранять в репозитории. Обычно это те, что не содержат программный код. Их можно 
исключить, указав в специальном файле .gitignore. Создаём такой файл
17. git status покажет появление этого файла. GIT предлагает додавить в индекс и отправить его в коммит. 
А также log, которвй был добавлен в папку logs
18. *.log, записанные в файле gitignore автоматически будут игнорить все файлы с расширением log
19. если файлы log находятся в оделной папке, то в .gitignore нужно написать logs/
20. если какой-то из файлов в папке игнорировать не следует, то logs/*.log, исключаем все файлы, лежащие в папке logs, а затем
не исключаем конкретный файл !logs/sample.log
